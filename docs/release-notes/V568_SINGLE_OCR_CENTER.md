# v568 — OCRセンター（一元実行）

## 方針
OCRを実行・停止・再試行・結果確認する画面を **OCRセンター** に集約しました。

- 既存のInbox OCR永続キューと共有フォルダの実行リースをそのまま利用
- Inboxの添付プレビューにはOCR実行操作を残さず、OCRセンターへの導線だけを表示
- OCRセンターで画像・PDFの追加、キュー操作、PDF本文確認、ページOCR、全ページOCR、結果確認、AIへの引き渡しを実施
- ページ、DB行、JournalへOCRキューを個別実装しないため、処理競合・設定分散を防止

## 制約
現行のOCRキューはInbox添付を正本にする設計です。Journalやページの添付をOCRしたい場合は、OCRセンターへ対象ファイルを追加して処理します。結果連携を各種添付へ戻す機能は、共通添付参照モデルを設計してから追加します。
