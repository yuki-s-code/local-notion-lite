# v638: ページ関係図（2D星座図）

## 目的
現在開いているページを中心に、ページリンク・バックリンク・親子ページ・タグ・データベース行を2Dの星座図として表示します。

## 性能設計
- 新しい全文索引・AI索引は作成しません。
- `workspace_link_index`、`pages`、`database_row_index`、`database_summary_index` のSQLite索引だけを使用します。
- 共有フォルダのページ本文、DB本文、OCR結果、埋め込みモデルは読み込みません。
- 1画面の上限は80ノード（APIでは20〜120に制限）です。
- ノード位置は端末内 `localStorage` に保存され、共有データには書き込みません。

## 操作
- サイドバーの `✦` または `Ctrl/Cmd + K` → 「現在のページの関係図」で開く。
- ホイールでズーム、背景ドラッグで移動、ノードドラッグで配置変更。
- ノードをクリックすると右側に概要を表示し、ダブルクリックまたは「開く」で対象を開く。
- リンク、バックリンク、親子、タグは左側で個別に表示切替できます。

## 関係の意味
- 実線: 現在ページからのリンク
- 破線: バックリンク
- 緑線: 親子ページ
- 点線: タグ
- DB行は既存のリソースリンク索引から取得します。

## 制約
これはローカルグラフです。全ワークスペースを星座図にする全体ナレッジマップは、フィルター・集約・負荷制御を別途設計してから追加します。
