# v715 Performance Attack

## 目的
v714で保存・作成・右パネルの常時負荷を減らした後の追加軽量化です。今回は、ページ数が増えたワークスペースで効きやすい「サイドバーツリーの描画負荷」と「定期同期の読み込み負荷」をさらに削りました。

## 変更点

- サイドバーのページツリーを仮想スクロール化しました。
  - 開いている階層を一度フラット化し、画面に見えている行だけを `TreeItem` としてマウントします。
  - 子ページは再帰描画せず、表示範囲外のDOMを作りません。
  - 子孫件数はキャッシュ付きで計算し、各行の `countDescendantPages()` 再帰を避けます。

- 定期軽量同期をさらに弱めました。
  - 3分ごとから5分ごとへ変更しました。
  - サイドバーが閉じている、またはツリー表示でない場合は tree も読みに行きません。
  - DB / Journal は該当画面を開いている時だけ同期します。

- `src` 配下の生成済み `.js` を削除しました。
  - `.ts` / `.tsx` 修正が古い `.js` に隠れる事故を避けます。

## 期待効果

- ページ数が多い環境でサイドバーの初期描画・折りたたみ・スクロールが軽くなります。
- ページ編集中の裏側I/Oがさらに減ります。
- 共有フォルダ環境での定期読み込みによる引っかかりを抑えます。
