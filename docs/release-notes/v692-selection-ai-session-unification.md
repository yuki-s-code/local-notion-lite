# v692 — 選択範囲AI編集セッションの統合

## 変更内容

`BlockNoteEditor.tsx` の選択範囲AI編集を、単一の `EditorAiSession` を正本とする構成へ変更しました。

- 選択範囲AI: `sourceText`、Tiptap範囲、ポップオーバー位置を1セッションとして固定
- ページAI: 開始時点のページ本文を別セッションとして固定
- 生成処理: `window.getSelection()` を参照せず、セッションの `sourceText` のみ送信
- 置換処理: 現在のブラウザ選択へフォールバックせず、セッションのTiptap範囲のみへ適用
- 置換前検証: 保存済み範囲の本文が変化している場合は、意図しない置換を防ぐため処理を中止
- 非同期応答: パネルを閉じた後・新しいセッション開始後に届いた旧リクエストの応答を破棄

## 解消する事象

- AI入力欄にフォーカスしたあと、選択範囲ではなくページ全体がAIへ送られる
- 以前の選択文が次のページAIに混入する
- 古い生成結果が、開き直したAIパネルへ表示される
- 選択が失われた後、別の位置へ置換される
