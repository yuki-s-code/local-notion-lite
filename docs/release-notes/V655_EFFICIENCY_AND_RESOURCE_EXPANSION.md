# v655 — 大量操作・添付Index・全体マップの効率改善

## 変更

- ページ階層の大量削除時、派生Indexを800件単位の集合SQLで削除。
- RendererのMutation購読を `subscribeWorkspaceMutations()` に共通化。
- 添付Indexの全件再構築を、進捗表示・中止可能・既存Index保持のジョブへ変更。
- 全体ナレッジマップは既定のページ中心を維持し、必要な時だけDB行／添付／Journalを上限付きで展開。

## 安全性

- 添付Indexは全ソース確認完了後に一度だけ置換するため、中止・SMB一時エラーで一覧が空にならない。
- DB行・添付の全体マップ展開はSQLite Indexのみを読み、共有フォルダ本文・OCR・Semanticを走査しない。
- 通常のページ中心マップは従来どおり軽量なまま。
