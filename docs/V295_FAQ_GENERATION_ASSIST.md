# v295 FAQ Generation Assist

## 概要

v295では、v294で追加した生成AIモデルフォルダ設定を使い、FAQ編集画面に「生成AIで改善案」を追加しました。

生成AIは最終回答を自動確定するためではなく、FAQを育てる補助として使います。

## 追加内容

- FAQ編集モーダルに「生成AIで改善案」セクションを追加
- `POST /smart-assist/faqs/improve-draft` を追加
- llama.cpp / GGUF が利用可能な場合はローカル生成を実行
- 生成AIがOFFまたは未準備の場合はテンプレート改善案を返す
- 改善案は自動保存せず、人が確認してから反映

## 生成される候補

- 改善質問
- 改善回答
- 想定質問
- 言い換え・短文キーワード
- 除外語 / negativeTerms
- 確認アクション
- 注意点

## 実務上の方針

生成AIは、制度内容・金額・期限・手続き名を勝手に作らない前提で使います。
根拠にない内容は「確認が必要」として扱います。

FAQ改善案は次のように利用します。

1. 既存FAQを開く
2. 「改善案を作成」を押す
3. 内容を確認する
4. 「検索ヒントだけ反映」または「本文も含めて反映」を選ぶ
5. 最後にFAQを保存する

## llama.cpp 実行

v294の生成AI設定で、次が揃っている場合のみ llama.cpp を使います。

- provider: `llama-cpp`
- enabled: true
- `.gguf` モデルファイル
- `llama-cli.exe` / `llama.exe`

未準備の場合でも、テンプレート改善案を返すため、UIは利用できます。
