# V362 AI Chat Answer Template Router

## 目的

v360の質問意図ルーターとv361の検索語展開・候補再ランキングを使い、右下AIの回答形式を質問内容に合わせて自動選択する。

これまでの右下AIは、質問意図や根拠状態を判定していても、生成プロンプトの出し方がまだ共通寄りだったため、以下のような課題が残っていた。

- 要約、手順、比較、文面作成、FAQで回答形式が混ざりやすい
- 小型LLMが「結論/根拠/確認事項」を繰り返しやすい
- 検索ベースfallbackでも質問に合った形になりにくい
- ユーザーから見て、AIがどの形式で答えようとしているか分かりにくい

## 追加内容

### 1. 回答テンプレート自動選択

`generateWorkspaceAiChatAnswer()` 内に `selectWorkspaceAiAnswerTemplate()` を追加した。

質問意図・回答モードに応じて以下を選択する。

- ページ要約
- TODO抽出
- 関連情報整理
- 手順化
- 比較・整理
- 文面作成
- FAQ形式
- 聞き返し
- 根拠重視
- 詳しい説明
- 短い回答
- スマート回答

### 2. 生成AIプロンプトへの明示

生成AIへ次を渡す。

- 回答テンプレート名
- テンプレートID
- 推奨構成
- テンプレート専用指示

これにより、同じ参照候補でも「手順として答える」「FAQとして答える」「文面として作る」などを明確に分ける。

### 3. UI表示

右下AIの回答に `回答形式: xxx` を表示する。

ユーザーは、AIが次のどの形式で回答したかを確認できる。

### 4. 次アクション候補の最適化

回答テンプレートに応じて、回答後の候補ボタンを出し分ける。

例:

- 手順化: `もっと短い手順にして` / `必要書類も出して`
- 文面作成: `もっとやさしく` / `職員向けに` / `保護者向けに`
- 比較: `違いだけにして` / `判断ポイントだけ出して`

## 期待効果

- 回答形式が質問内容に合いやすくなる
- 小型LLMの不要な固定見出し・反復を抑えやすくなる
- ユーザーが回答の意図と形式を理解しやすくなる
- 右下AIが「検索して答える」だけでなく、要約・手順化・文面化・FAQ化まで自然に切り替わる
