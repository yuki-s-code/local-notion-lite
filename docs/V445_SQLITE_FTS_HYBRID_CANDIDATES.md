# V445: SQLite FTS5ハイブリッド候補補強

## 目的

sqlite-vecの意味候補抽出を維持しつつ、規程名、年度、受付区分、正式名称、タグなどの明示的な語を含む根拠を候補集合へ補強する。

## 構成

- `workspace_semantic_vec_*`: sqlite-vecによる意味近傍候補
- `workspace_semantic_fts`: SQLite FTS5 `trigram` によるタイトル・本文・メタ情報の一致候補
- 既存のJavaScript再順位付け: タイトル・本文・タグ・関係性を評価

FTS5は検索結果をそのまま確定しない。sqlite-vecの候補群へ追加するだけであり、最終順位は従来の安全な再順位付けを通す。

## 同期

- ローカルSQLiteキャッシュだけに作成する。
- 共有フォルダの正本JSONには保存しない。
- テキストハッシュが変わらないチャンクはFTS行を再作成しない。
- 削除済みチャンクだけFTS行を削除する。
- FTS5/trigramを利用できないSQLite環境では、sqlite-vec既定検索を維持して自動で機能を省略する。

## 運用

更新後に一度だけ「差分更新」または「全件再生成」を実行する。管理画面でFTS5の同期件数がSemantic Index件数と概ね一致し、検索時に「FTS補強回数」が増えれば有効である。
