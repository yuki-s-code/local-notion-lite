# v51 Adaptive Editor Layout

## Goal
BlockNoteの本文幅が、サイドバー・右サイドピーク・Pinned database表示時に極端に狭くならないように、固定幅中心のレイアウトを可変幅中心へ修正しました。

## Changes
- サイドバー幅を `clamp()` で可変化
- 本文左右余白を画面幅に応じて可変化
- BlockNote本文の最大幅を 860px 固定から最大 980px / 100% に調整
- Pinned database と本文を同時表示できる場合は2カラム、狭い場合はDBを上部固定へ自動移動
- 右サイドバー表示時はドロワー幅と本文余白を縮小して、本文が潰れにくいように調整
- サイドバー非表示時は本文に広い執筆スペースを割り当て
- データベース画面はページ本文の最大幅制約から外し、広いテーブル表示を優先

## Notes
Pinned databaseは画面幅が足りない場合、右カラムではなく本文上部のstickyカードとして表示されます。これにより、長文編集時もBlockNote本文幅を優先できます。
