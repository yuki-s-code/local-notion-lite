# v207 Final Ranking Sort Fix

## 目的

v206 では、候補差分補正によって暫定1位のスコアだけが下がった場合でも、候補配列を再ソートしていませんでした。
そのため、表示上は候補2が96%なのに、回答本文は候補1の53%のFAQから生成される不整合が発生する可能性がありました。

## 修正内容

- Guarded Ranking の最終スコア確定後に候補を再ソート
- 回答に使用するFAQと候補表示の1位を一致させる
- `__forceShortQueryMargin` など内部用プロパティを最終候補から除去
- 同点時は質問文で安定ソート

## 期待される改善

例: `クラブの費用を教えて`

修正前:

- 回答本文: 放課後児童クラブの減免制度FAQ
- 候補1: 減免FAQ 53%
- 候補2: 費用FAQ 96%

修正後:

- 回答本文: 放課後児童クラブの費用FAQ
- 候補1: 費用FAQ 96%
- 候補2: 減免FAQ 53%

## 注意

この修正はランキング確定処理の不整合修正です。Transformers.js、semantic-index、kuromoji + n-gram 検索、Guarded Ranking の基本構成は維持しています。
