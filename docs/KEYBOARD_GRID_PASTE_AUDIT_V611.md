# V611 — テーブルのキーボード操作・複数セル貼り付け

## 実装
- `Tab` / `Shift+Tab`: 編集可能セル間を移動（行端で次/前の行へ折返し）
- `Enter`: 同じ列の次行へ移動
- Excel/スプレッドシートからのタブ・改行を含む貼り付け: 現在セルを左上として一括適用
- チェック列は `true / 1 / 完了 / 済` 等をチェック済みに変換
- 数値列はカンマを除去して数値化。無効な数値は更新しない

## 性能設計
- キー/ペーストのイベントはテーブル要素の委譲イベントのみ。各セルに新しいリスナーを作らない。
- 複数セル貼り付けは変更セルを `Map` に集約し、1回の `mutate` と1回の保存予約で処理。
- 最大2,000セルに上限を設定。大きすぎる貼り付けによるRenderer固まりを防止。
- Server Tableでは、現在読み込み済みのページ内だけを対象にする。未読の全行をRendererへ取り込まない。
- Relation、Multi-select、Formula、Rollup、Unique ID、自動日時は一括貼り付け・グリッド移動対象から除外し、データ破損を防止。

## Indexへの影響
- 通常のDB保存と同一の単一更新として処理されるため、既存のDB行差分IndexおよびSemantic差分更新キューが利用される。表示設定やレイアウト設定は変更しない。
