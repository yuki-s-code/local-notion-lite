# V373 保存フロー完全統一

## 目的

画面遷移、右サイドピークの閉じる操作、Journal、FAQ編集、アプリ終了時に、
デバウンス中または保存キュー内の変更が失われないようにする。

## 実装

- 通常ページ / DB / Journal / FAQの保存を最後の状態を優先する直列キューへ寄せた。
- Inbox、Home、Smart Assist、管理画面などへの遷移前に `flushPendingSaves()` を実行する。
- 右サイドピークの × でも保存完了を待って閉じる。
- Electron終了時はrendererに保存要求を送り、最大5秒待ってからローカルAPIとSQLiteを停止する。

## 制約

- 強制終了、Windowsの強制再起動、電源断では保存完了を保証できない。
- 共有フォルダ自体が一時的に利用不能な場合、画面上に保存失敗を表示し、未保存状態を維持する。
