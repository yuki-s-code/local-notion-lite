# V421: タグ候補の一括レビュー

## 目的

ページ単位で候補を確認する既存のタグ候補機能に加え、ワークスペース内の複数ページをまとめて確認し、利用者が選んだタグだけを一括で追加できるようにした。

## 画面

`タグ管理` の上部に **「タグ候補をまとめて確認」** を追加した。

1. `候補を確認` を押す。
2. 全ページを最大3件ずつ並列に読み込み、既存タグだけを候補として評価する。
3. タイトル・現在のタグ・候補と根拠を確認する。
4. 不要な候補のチェックを外す。
5. `選択した候補を追加` を押す。

候補の確認中は進捗が表示され、`確認を中止` で残りの読み込みを止められる。

## 判定根拠

候補生成は既存の `suggestTagsFromContent` を共通利用する。

優先順位:

1. タイトル・本文にタグ名が直接一致
2. 登録済みの別名・表記ゆれに一致
3. 現在のページタグと、他ページで頻繁に併用されるタグ
4. タグの利用ページ数（弱い同点調整のみ）

候補は既存タグに限定される。新しいタグをAIが作ることはない。

## 保存安全性

- スキャンは明示操作時だけ行う。起動・画面表示時の先読みはしない。
- スキャンは最大3並列で実施し、共有フォルダへの瞬間的な負荷を抑える。
- 一括追加前に、対象ページを1件ずつ再取得する。
- 保存には再取得した `updatedAt` を `baseUpdatedAt` として渡すため、候補表示後に他端末で更新された内容を古い情報で上書きしない。
- 更新対象は `properties.tags` のみ。本文、コメント、BlockNoteデータ、ページ履歴は変更しない。
- 保存に失敗したページがあっても、他ページの追加処理は継続し、結果をステータスに表示する。

## 制約

大量ページのワークスペースでは初回スキャンに時間がかかる。これは全ページの本文を正確に判定するためであり、意図しないバックグラウンド先読みは行わない。
