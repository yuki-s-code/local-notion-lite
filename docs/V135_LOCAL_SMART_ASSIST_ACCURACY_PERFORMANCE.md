# V135 Local Smart Assist Accuracy / Performance

## 目的
OllamaやクラウドAIを使わず、完全ローカルのまま Local Smart Assist の検索精度・候補精度・処理効率を上げる。

## 追加・改善

- 正規化強化
  - NFKC正規化
  - カタカナをひらがなへ寄せる
  - 空白ゆれを吸収

- 形態素解析風トークン強化
  - 漢字・かな・英数字の抽出
  - 2-gram / 3-gram
  - ストップワード除去
  - 類義語展開

- 検索精度改善
  - Fuse.js対象に tokenText を追加
  - タイトル・本文・タグ・トークンを重み付け
  - 検索語と文書のトークン重なりをスコアに反映
  - 日付・優先度・状態シグナルを検索対象へ追加

- Relation候補改善
  - トークン重なり
  - タイトル類似
  - タグ一致
  - 同日ボーナス
  - 同種別ボーナス
  - 候補理由を表示

- TODO抽出改善
  - 対応、確認、提出、レビュー、承認などの表現を追加
  - 不要・対応済みなどの誤検出を抑制

- 重要文抽出改善
  - 重要語、日付、期限、箇条書き、位置をスコア化
  - 長すぎる文や短すぎる文のノイズを抑制

- 効率化
  - 検索入力を160ms遅延反映
  - 文書ごとの tokenSet / tokenText / searchText を事前生成
  - 類似度計算で再トークン化を減らす

## 注意
この機能はLLMではなく、ローカルのルール・検索・スコアリングによる Smart Assist です。文章生成AIのような自由回答は行いませんが、容量をほぼ増やさずに検索・整理・候補提示を強化します。
