# v374 起動・遷移・ロック安定化

- ウィンドウの ×、Alt+F4、メニュー終了を同じ shutdownApplication() に統一し、Renderer の保存 flush を最大5秒待ってから API / SQLite を停止する。
- 起動設定は one-shot の `app:ready` だけに依存せず、Renderer が `app:getBootstrap` を invoke して取得する。
- 起動時は一覧を先に描画し、共有フォルダ取込は120ms後にバックグラウンド開始する。
- Journal / DB / サイドピークにもナビゲーション番号を適用し、遅い古い読み込み結果で画面を戻さない。
- 右サイドピークは閲覧専用にした。メイン編集とのロック所有権を混ぜないためで、編集は「メインで開く」から行う。
