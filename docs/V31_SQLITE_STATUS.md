# v31 SQLiteステータス表示の修正

## 修正内容

- `/health` APIでSQLiteの状態、保存先、ファイル名を明示的に返すようにしました。
- 共有フォルダ同期が一時的に失敗しても、SQLite表示が `-` に戻らないようにしました。
- UI側では `SQLite: -` ではなく、取得中は `SQLite: 確認中`、取得後は `SQLite: 利用中` と表示します。
- `local.sqlite` の実パスはツールチップで確認できます。

## 背景

以前は `reload()` 内で共有フォルダ同期・ページ一覧・DB一覧・ヘルスチェックをまとめて実行していました。
そのため、共有フォルダ同期が遅い場合や一部で失敗した場合、SQLiteの状態が取得されず `-` 表示になることがありました。

v31ではSQLite状態取得を独立させています。
