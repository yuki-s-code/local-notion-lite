# v418: 共有タグ別名の同時編集対策

`workspace/tag-aliases.json` に `revision` を追加し、クライアントが読み込んだ辞書のスナップショットを保存時に送信します。

- 異なるタグを別端末で編集した場合: サーバーが差分を自動統合します。
- 同じタグを別端末で異なる内容に編集した場合: 競合として返し、共有ワークスペース側の内容を保持します。
- v417クライアントが接続した場合: 後方互換の加算マージで既存別名を消さないようにします。
- 既存のversion 1 JSON: 初回読込み時はrevision 0として扱い、次回保存でversion 2へ移行します。

この実装は共有フォルダ上のロック機構と原子的書込みを使用します。
