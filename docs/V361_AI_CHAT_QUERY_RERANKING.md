# V361 AI Chat Query Expansion / Candidate Reranking

## 目的

右下AIチャットの回答前に、Semantic Search の結果をそのまま使うのではなく、質問に直接答えている候補を優先するための再ランキングを追加しました。

## 変更内容

- 質問文を内部で言い換え検索
  - 学童 → 放課後児童クラブ / 入会 / 費用 / 減免 / 延長 / 見学予約
  - 料金 → 費用 / 利用料 / 月額 / おやつ代 / 延長利用料
  - 申請 → 申請方法 / 手続き / 提出先 / 期限 / 必要書類
  - 予約 → 見学予約 / 受付 / 予約方法 / 対象施設
- 質問そのものと、言い換え済み検索語の検索結果を統合
- FAQ質問ではFAQを優先検索
- 手順質問ではFAQ/Page/DB行を優先検索
- 候補ごとに以下を評価
  - タイトルが質問語に一致しているか
  - 本文が質問語に一致しているか
  - メタ情報が質問語に一致しているか
  - 質問意図に合う種類か
  - 参照元として固定されているか
- タイトルだけ近く、本文根拠が弱い候補は少し下げる
- 回答文脈に `question_terms` と `answer_fit` を渡す
- UIの根拠表示に `適合` スコアを表示

## 効果

- 「雰囲気だけ近い候補」が上に来にくくなる
- FAQ/手順/比較など、質問意図に合う候補が上がりやすくなる
- 参照元カードの選定が回答に直結しやすくなる
- 根拠が弱い候補による不自然な回答を抑える

## 注意

完全なLLMリランキングではなく、ローカル軽量実行向けのルールベース再ランキングです。会社端末での速度を維持するため、追加の重いモデルは使っていません。
