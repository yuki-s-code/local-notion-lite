# V327 Semantic Index Idle Diff Update

## 目的

Semantic Indexの差分更新を、ユーザー操作がない時間に少量ずつ自動実行できるようにしました。
保存時にはembedding生成を走らせないため、ページ・DB・Journal保存の体感速度を守ります。

## 追加内容

- Workspace Semantic管理画面に「アイドル時の自動差分更新」を追加
- ON/OFF、1回の更新件数、待機秒数を設定可能
- 設定はTransformer設定に保存
- 操作が一定時間ない場合のみ `POST /semantic/diff-update` を少量実行
- 更新待ちが0件の場合は何もしない
- 手動更新中・他の処理中は自動更新しない

## 推奨設定

- ON
- 1回の件数: 10件
- 待機秒数: 8秒

大量取込後は手動の「差分更新 100件」を使い、日常運用では自動更新に任せる構成を推奨します。
