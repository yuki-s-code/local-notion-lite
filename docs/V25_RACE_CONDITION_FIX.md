# v25 Race Condition Fix

ページクリック時に本文が表示されない、別画面や別ページの内容が残る問題への対策。

## 原因

- ページAの読み込み中にページBをクリックすると、ページAのAPI結果が後から返り、`current` を上書きする場合があった。
- 自動保存の完了処理がページ切替後に返ると、古いページの `saved` 結果で `current` やBlockNote用stateを上書きする場合があった。
- BlockNoteは内部描画状態を持つため、保存完了のたびに本文stateをサーバー応答で再注入すると、入力中の描画と保存応答の順序がずれて表示が戻ることがあった。

## 対策

- `pageOpenSeqRef` を追加し、古いページ読み込み結果を破棄。
- `saveSeqRef` と `currentPageIdRef` を追加し、古い保存結果を破棄。
- ページ切替、DB表示、ゴミ箱表示で未完了のページ読み込み/保存を無効化。
- 同じページの保存完了時にBlockNote本文を再初期化しない。
- ページ本文の正本は保存リクエスト時点のスナップショットを送信し、描画中のBlockNoteインスタンスは維持する。

## 期待する挙動

- サイドバーでページを連続クリックしても、最後にクリックしたページだけが表示される。
- データベースやゴミ箱からページへ戻っても、古い画面が残らない。
- 自動保存完了後に本文が前の状態へ戻りにくくなる。
