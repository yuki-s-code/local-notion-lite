# V433: ページ初回表示のBlockNote遅延マウント

## 目的
ページを開く瞬間にBlockNote/Tiptapとカスタム拡張の初期化が集中し、本文が見えるまで待たされる問題を緩和します。

## 変更
- `BlockNotePageEditor` に `deferEditorMount` を追加。
- 通常のページ本文では初回描画を優先し、最大180msのアイドル時間までBlockNote/Tiptapの生成を遅延。
- 遅延中は本文ブロックから作った軽量なテキストプレビューを表示。
- エディタはページID単位で初期化するため、ページ切替時に前ページのエディタが残らず、先に新ページのプレビューを表示します。
- ジャーナル、リンクプレビュー、データベース行のエディタには適用しません。既存の即時編集挙動を維持します。

## 効果
- 初回ページ遷移で本文領域が空白になる時間を短縮。
- BlockNote初期化と、v429/v430/v432で遅延化した関連情報・共有同期との競合を減少。
- すぐページを切り替えた場合、不要になった遅延初期化はキャンセル。

## 非変更点
- ページ本文、ブロックデータ、タグ、履歴、保存形式は変更しません。
- 編集操作、BlockNoteのカスタムブロック、スラッシュメニュー、添付アップロードは従来どおりです。
