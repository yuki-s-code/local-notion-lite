# V431: 正確な遅延サイドパネル件数

## 目的
V430でコメント・履歴・リンク詳細を遅延読込にした結果、未読込データを `0` 件に見せないようにする。

## 変更
- `GET /pages/:id/sidebar-counts` を追加。
- コメント未解決数、履歴スナップショット数、バックリンク数だけを返す。競合数は履歴タブを開いた時だけ確認する。
- 履歴件数は履歴ディレクトリ数のみを数え、各 `meta.json` を読み込まない。
- バックリンク件数は SQLite の `COUNT(*)` を使い、リンク行一覧を転送しない。
- 詳細一覧は従来どおり、対応タブを開いた時だけ取得する。
- ページ表示後120msで件数だけ取得する。取得前はタブに `…` を表示する。
- コメント更新・履歴作成・詳細タブ読込後は、件数だけを再取得する。
- 履歴タブを開いて競合が見つかった場合は、履歴タブに `競合 N` を表示する。

## データ量と安全性
この変更はページ本文、タグ、履歴、コメント、リンクの保存形式を変更しない。件数APIは読み取り専用である。
