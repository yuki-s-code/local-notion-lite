# V442: 編集中の仮関連候補

## 目的
ページの未保存本文から関連候補を表示し、編集中でも参考ページを確認できるようにする。

## 動作
- ページが編集状態かつ未保存変更がある時だけ有効。
- 入力停止から2.5秒後に、最大4,000文字の未保存本文・タイトル・タグを検索語として既存Semantic Indexへ問い合わせる。
- 最大5件を「編集中の候補」として別枠に表示する。
- 仮候補は共有フォルダ、ページ本文、履歴、Semantic Index、Embeddingキャッシュを書き換えない。
- 入力が再開・ページ切替した場合、前のタイマー／結果は破棄する。
- 保存後は既存の低優先度差分Index更新が正式結果を更新する。

## 実務上の意図
入力ごとにIndex再構築・Embedding保存を行うと、長文や低性能端末で編集が重くなる。本機能は既存Indexへの読取り検索だけに限定し、正式Index更新とは分離する。
