# v440 Semantic Index 失敗管理・対象別再Index

## 目的
長文、画像埋め込み、破損したBlockNoteデータ、モデル一時エラーなどで一部のSemantic IndexチャンクがEmbeddingできない場合に、全件再生成をせずに原因を確認・復旧できるようにする。

## 追加した運用機能

- ローカルSQLiteキャッシュに `workspace_semantic_failures` を追加
- 失敗したチャンクについて、対象ID、ページ名、種別、チャンク位置、最終失敗日時、エラー概要、失敗回数を保持
- Smart Assist 管理画面に **Index失敗・再Index** セクションを追加
- ページ型の失敗は「ページを開く」から原文を確認可能
- **この対象だけ再Index** は、選択した `sourceId` の全チャンクだけを強制再Embeddingする
- 他のページ、FAQ、DB行、Journalの既存Embeddingは維持する
- 対象再Indexが成功したチャンクは失敗一覧から自動的に消える

## 運用手順

1. Smart Assist 管理画面で失敗件数を確認する
2. 失敗項目のページを開き、長大なBase64、未保存状態、極端に長いプロパティなどを確認する
3. 必要ならページを保存する
4. **この対象だけ再Index** を実行する
5. 成功後、失敗一覧から消えたことと関連ページ表示を確認する

## 失敗が続く場合

- モデルの準備状態、ローカルキャッシュの空き容量、メモリ不足を確認する
- 対象ページの画像埋め込み・巨大な貼り付けデータを確認する
- 同じ失敗が繰り返される場合だけ、エラー概要を控えて原因調査する
- 全件再生成は、モデル変更、Index形式変更、キャッシュ破損時に限定する

## 性能方針

対象再Indexは失敗したページのチャンクだけを再Embeddingする。ページ全体の再走査や全件のEmbeddingを避けるため、通常作業中の負荷を最小化する。
