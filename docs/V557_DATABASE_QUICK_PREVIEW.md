# v557: ページ／データベースのクイックプレビュー

## 目的
- 既存のページツリーのホバープレビューを維持しつつ、データベースとデータベース行にも同じ「開く前に確認する」導線を追加する。
- ワークスペースの既存タブを不必要に切り替えず、共有フォルダの読み込み回数も増やさない。

## 実装
- サイドバーのデータベース名にポインタを置くと、ポータル型のプレビューを表示する。
  - 共有／個人スコープ
  - 更新日時
  - 行数、プロパティ数、ビュー数
  - 先頭5件のプロパティ名
  - 現在サイドバーで保持している最近の行（最大4件）
- 展開済みのデータベース行にポインタを置くと、行名、更新日時、子ページ数を表示する。
- プレビュー自体はクリックを奪わず、DB／行を開く操作は従来どおり各サイドバー項目のクリックで行う。
- 幅980px以下ではポータルプレビューを表示しない。狭い画面で本文を隠さないため。

## パフォーマンス／共有フォルダ
- プレビューのための新しいAPI呼び出しは行わない。
- サイドバーがすでに取得しているDBメタデータと行一覧だけを使用する。
- DBが未展開の場合も、DB本体に保存済みの行データから最大4件を表示する。
