# V377 保存失敗時の復旧

v377では、通常ページ・データベース・Journal・FAQの保存失敗時に、最後の未保存スナップショットを保持します。

## 自動再試行

共有フォルダの一時的なI/O遅延などで保存が失敗すると、次の間隔で再試行します。

1. 2秒後
2. 5秒後
3. 10秒後

3回を超えて失敗した場合は自動再試行を止め、画面上に **未保存 ○件・再試行** または **FAQ未保存・再試行** を表示します。ユーザーは保存内容を失わず、共有フォルダの復旧後に手動で再試行できます。

## 競合・ロックエラー

DBの `DATABASE_CONFLICT` と `DATABASE_LOCKED` は一時I/O障害ではないため、読み取り専用に切り替えた上で既存の競合メッセージを優先します。保存データはキューに残しますが、ユーザーが競合を確認してから再試行します。

## 範囲

- 通常ページ本文・タイトル・プロパティ
- データベースの保存キュー
- Journal
- Smart Assist FAQ

添付のアップロード、ページ削除、バックアップなどの一回限りの操作は対象外です。失敗時は既存のエラーメッセージを確認して再実行してください。
