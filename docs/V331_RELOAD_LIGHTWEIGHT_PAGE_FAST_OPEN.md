# V331 reload軽量化・ページ即時表示

## 目的

会社端末・共有フォルダ運用で体感速度を改善するため、画面遷移時に重い全件再読込を避け、ページ本文を先に表示する。

## 変更点

- `reload()` を内部的に `refreshWorkspace()` ベースへ整理
- 軽量更新用の `lightRefresh()` を追加
- 管理・手動同期用の `fullMaintenanceRefresh()` を追加
- 30秒ごとの自動同期を、共有フォルダ全体再取込なしの軽量同期に変更
- ページを開く処理で、本文・タイトル・BlockNote内容を先に表示
- 添付・履歴・競合・バックリンク・コメント・アクティビティは遅延読込化
- ロック取得後の全件reloadを廃止し、軽量更新へ変更
- 保存後の全件reloadを廃止し、軽量更新と周辺情報の非同期再取得へ変更
- 手動同期ボタンは引き続きフルメンテナンス更新として利用

## 期待効果

- ページクリック直後の反応が速くなる
- バックリンクや履歴の読み込み待ちで本文表示が止まらない
- 30秒自動同期による共有フォルダI/O負荷を軽減
- 保存後に画面全体が重くなる症状を軽減

## 注意

- ゴミ箱復元、完全削除、履歴復元、データベース保存など、全体整合性が重要な操作では従来の `reload()` を残している
- `reload()` はフル更新用途として残し、軽量化対象は画面遷移・保存・ロック更新に限定している
