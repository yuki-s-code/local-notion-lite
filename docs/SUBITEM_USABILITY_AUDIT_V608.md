# V608 サブアイテム操作性・全体整合監査

## 実装

- 親行の番号列に、子行を表示／折りたたむトグルを追加。
- 折りたたみ状態は `localStorage` の `fast-db-subitems-collapsed:<databaseId>` に端末単位で保存。
- 親行には、現在のフィルター結果内の直接の子アイテムについて `完了数/件数` を表示する。
- 折りたたみ操作は、Relationセル、DB行、行順、Semantic Index、リンクIndex、タスクIndexを更新しない表示専用操作。
- 親子階層は引き続き行IDを一度しか出力しないため、既知の重複表示・循環関係対策を維持。

## 意図した範囲

- StatusがあるDBのみ、値が正確に `完了` の直接の子を完了として集計する。
- フィルターにより表示対象外の子は、Notionの表示結果と同様に進捗表示の分母に含めない。
- サブアイテム有効時は全親子の順序が必要なため、既存どおりServer Tableページングを使わない。

## 次の候補

1. Statusに「完了グループ」というメタデータを持たせ、任意の完了名に対応する。
2. 親の進捗を専用のRollupプロパティとして保存・フィルター・集計可能にする。
3. 依存関係プロパティとガント依存線を追加する。
