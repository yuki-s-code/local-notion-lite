# V565 – Journal競合解決画面

## 目的
共有フォルダ上で同じ日付のJournalを複数端末または複数ウィンドウから更新した場合に、未保存内容を失わず、利用者が保存方法を選択できるようにする。

## 実装
- `JOURNAL_CONFLICT` の自動再試行を停止し、競合解決モーダルを表示。
- 自分側と共有フォルダ側を本文・気分・天気付きで並列表示。
- 操作: 共有側を読み込む / 本文を統合して保存 / 自分側で上書き。
- 競合を返す前に、サーバー側でローカル送信内容を `backups/journal_conflict_*` に保存。
- モーダル表示中はJournalの自動保存を停止し、同じ競合の反復を防止。

## 注意
本文統合は、共有側本文の後に区切り線とこの端末の本文を追加する安全な追記型。細かな行単位統合は、後続の差分編集機能で改善可能。
