# V352 llama-server ctx診断・推奨設定表示

## 目的
v350の右下AIチャットは、読込モードに応じて必要なcontext sizeが変わります。
ただし、既に起動済みのllama-serverが小さいctxで動いている場合、1000文字以上のページ要約や詳細モードで生成に失敗することがあります。

v352では、設定ミスを防ぐために以下を追加しました。

## 追加内容

- 右下AIチャットに推奨ctx表示を追加
  - 高速: ctx 2048
  - 標準: ctx 4096
  - 詳細: ctx 6144以上
- 右下AIチャットにllama-server状態表示を追加
  - 起動状態
  - 現在ctx
  - 使用メモリ
- 右下AIチャットに「推奨ctxで起動/再起動」ボタンを追加
- 生成AI設定画面にも常駐ctx / 推奨ctx / ctx診断を追加
- アプリが起動したllama-serverについてはctxを状態として保持
- 生成失敗時にctx不足の可能性がある場合、警告文に推奨ctxを表示
- `/smart-assist/generation-server/start` が `contextSize` と `forceRestart` を受け取れるように変更

## 方針

v351のような安全短縮prompt再試行は入れず、v350を基準にしています。
今回の目的は、長文生成そのものの仕様変更ではなく、llama-serverのctx設定ミスを見つけやすくすることです。

## 注意

外部で手動起動したllama-serverの場合、ctxを正確に取得できないことがあります。
その場合は右下AIに「ctx不明」と表示し、読込モードに合うctxで手動起動するよう案内します。
