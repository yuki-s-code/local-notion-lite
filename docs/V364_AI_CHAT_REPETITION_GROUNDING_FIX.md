# V364 AI Chat Repetition / Grounding Fix

## 目的
右下AIチャットで、生成AIが同じ確認事項を繰り返したり、参照元にない予約方法（メール・インターネット・書類など）を補完してしまう問題を抑制する。

## 主な変更

- 手順回答のプロンプトを強化
  - 最大6項目まで
  - 参照元にない手段・期限・必要書類を作らない
  - 参照元に電話しかない場合は、メール・インターネット・書類などを推測しない
  - 同じ見出し・同じ確認事項を繰り返さない

- 手順回答の生成上限を調整
  - 高速: 最大320 tokens
  - 標準: 最大420 tokens
  - 詳細: 最大520 tokens

- 生成後の整形を強化
  - 同一行の重複除去
  - 同一セクションの重複除去
  - 参照元にない予約手段を含む行を除去
  - 番号付き項目が増殖した場合は安全な参照元ベース回答へ切替

- 根拠検証と連動
  - 手順回答に参照元外の断定が混ざった場合、生成AI回答を出さず、参照元ベースの安全な手順回答へ切替

- llama.cppの反復抑制
  - llama-serverの`repeat_penalty`を追加
  - llama-completionにも`--repeat-penalty 1.15`を追加

## 期待される効果

「見学方法について教えて」のような質問で、参照元に電話予約しかない場合、メール・インターネット・書類予約などを勝手に列挙し続ける挙動を抑制する。
