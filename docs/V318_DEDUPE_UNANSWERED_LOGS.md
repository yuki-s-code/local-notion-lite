# v318: 未回答・要改善ログの重複表示修正

## 修正内容

v316/v317で、未回答・要改善ログは以下を統合表示するようになりました。

- 低信頼チャットログ
- 改善キュー
- 「違う」フィードバック

このため、同じ質問が「違うフィードバック」と「改善キュー」の両方に保存され、画面上で2件ずつ表示されることがありました。

## 対応

- サーバー側の `listLowConfidenceSmartAssistLogs()` で同一質問を集約
- `feedbackId` が一致する改善キューは、対応するフィードバック表示に統合
- 同一質問・同一FAQ候補の重複は1件だけ表示
- 表示優先度は `feedback > queue > low-confidence-log`
- 改善キュー保存時も、質問・理由・FAQ候補が同じものは集約
- フロント側でも念のため表示前に重複排除

## 期待結果

「違う」を押したあと、未回答・要改善ログには同じ質問が2件ずつ出ず、1件にまとまります。
