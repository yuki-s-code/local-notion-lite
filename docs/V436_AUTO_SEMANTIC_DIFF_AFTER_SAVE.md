# V436 保存後の優先Semantic差分更新

ページ保存後、6秒の操作停止を待ってからSemantic Indexの差分更新を自動実行します。

- 保存中の本文・タグ編集をまとめて1回に集約
- 保存したページの `page:<id>` チャンクを、差分キューの先頭でEmbedding化
- 更新完了後は `local-notion:semantic-index-updated` を通知し、現在の関連ページパネルだけを自動再検索
- 保存直後は古い関連結果を確定結果として扱わず、「関連Indexへ反映しています」と表示
- 差分更新失敗時はページ保存を失敗扱いにせず、次回保存・手動差分更新で再試行

共有正本の保存処理とEmbedding更新は分離されているため、低速な端末や共有フォルダでも編集・保存操作を待たせません。
