# V439 編集優先のSemantic Indexジョブ制御

## 目的
本文編集、保存、Smart Assistの対話を、低優先度のSemantic Index差分更新より常に優先する。

## 動作
- BlockNote本文、タイトル、ページプロパティの編集開始時に、バックグラウンドIndex処理へ10秒の待機を通知する。
- 通知は最大約0.9秒に1回へ抑制し、入力ごとの通信を避ける。
- 保存後の自動差分更新は、編集が10秒止まるまで開始しない。
- すでに実行中の低優先度Index更新は、現在のEmbedding 1件を完了した後、次のチャンク開始前に非同期で待機する。
- 手動の全件再生成・手動差分更新・Smart Assist検索はbackground指定を使わず、待機の対象にしない。

## 制約
Transformers.jsの1回のEmbedding呼び出し自体は途中キャンセルしない。途中キャンセルはモデル状態やファイル書込みを不安定にするためであり、最大でも現在の1チャンクが終わった後に停止する。

## 結果
長文ページのチャンク更新中に編集を再開しても、本文編集と画面操作を優先し、差分更新は編集停止後に自動再開する。
