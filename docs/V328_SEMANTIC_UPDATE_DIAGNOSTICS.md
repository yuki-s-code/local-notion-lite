# v328 Semantic Index 更新履歴・診断

## 目的
v327で追加したアイドル時自動差分更新を、実運用で確認・診断しやすくするための更新です。

## 追加内容

- Workspace Semantic SQLiteに `workspace_semantic_runs` を追加
- 差分更新・全件再生成の実行履歴を最大50件保持
- 管理画面に更新履歴・診断カードを追加
- 直近の更新モード、生成件数、再利用件数、残り件数を表示
- 失敗・一部更新時のエラー概要を表示
- キャッシュ診断情報に `diagnostics: v328-run-history-and-safe-retry` を追加

## 運用

通常は「差分更新 20件」またはアイドル時自動更新を使います。
検索結果が古いと感じた場合は、まず差分更新を実行してください。
モデル変更、キャッシュ破損、大量の構造変更があった場合のみ全件再生成を使います。

## 方針

更新履歴は診断用です。履歴の保存に失敗しても、Semantic Indexの更新自体は止めません。
