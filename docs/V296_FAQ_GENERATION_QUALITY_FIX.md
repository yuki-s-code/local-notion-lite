# v296 FAQ Generation Quality Fix

## 目的
v295では、生成AIを設定しても改善案が元FAQとほぼ同じに見える場合があった。

主な原因は次のどちらか。

- llama.cpp の出力がJSONとして解析できず、テンプレート改善案に近い表示になっていた
- 生成モデルが保守的に元質問・元回答をそのまま返していた

## 修正内容

- FAQ改善プロンプトを強化
  - 元質問・元回答をそのままコピーしないよう明示
  - 回答は「結論 → 確認事項 → 注意点」の形に整理するよう指示
  - likelyQuestions は代表質問と重複しない別表現に限定
- 生成結果の品質チェックを追加
  - 質問類似度
  - 回答類似度
  - JSON解析可否
  - モデル名
  - 生成時間
- 生成結果が元FAQとほぼ同じ場合の補正を追加
  - 検索ヒント
  - 言い換え
  - 確認アクション
  - 注意メッセージ
- UIに「生成確認」を追加
  - llama.cppを使ったか
  - どのモデルか
  - 元FAQとの類似度
  - JSON解析できたか
- 生出力を折りたたみで確認可能にした

## 設計方針
生成AIはFAQを自動確定しない。
実務では、人間が改善案を確認して反映する。
