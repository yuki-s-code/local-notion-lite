# v128 Database Auto Save and Computed Value Check

## 目的

BlockNoteエディターと同じ感覚でデータベース編集も自動保存できるようにし、v127で追加したRollup / Formulaの計算を再点検しました。

## 自動保存

- 通常のデータベース画面でも、セル編集・行追加・列追加・ビュー変更・プロパティ変更後に自動保存します。
- 既存のDatabaseTable内部の900msデバウンスを利用します。
- 即時保存が必要な操作、例: 行追加、ビュー削除、プロパティ削除などはこれまで通りimmediate保存します。
- 保存状態はDBタイトル下に「編集中 / 保存中 / 保存済み」と表示されます。

## Rollup再点検

以下を修正しました。

- `Boolean(value)` 判定を廃止し、checkboxとして安全に判定する `isCheckedDatabaseValue` を追加。
- 文字列 `false` が完了扱いになる問題を防止。
- `percent_checked` の分母を対象行数として明確化。
- 数値Rollupは `1,000` や `80%` 形式も数値として扱えるように `toDatabaseNumber` を追加。
- 平均・合計は小数2桁丸めに統一。

## Formula再点検

以下を修正しました。

- `progress(完了数, 全体数)` でRollupの `67%` のような文字列が入っても数値化できるようにしました。
- `{プロパティ名}` の数値式でも、カンマ付き数値やパーセント表記を数値として扱います。
- Formula同士の循環参照は引き続き無効化しています。

## 注意

RollupとFormulaは保存データとしてセルに固定せず、表示時に計算します。これによりRelation先の変更が反映されやすくなります。
