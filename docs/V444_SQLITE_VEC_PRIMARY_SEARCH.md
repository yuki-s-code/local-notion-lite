# V444 sqlite-vec 既定検索

## 方針
- sqlite-vecが利用可能な端末では、Semantic Searchの候補抽出をsqlite-vecへ固定。
- タグ・本文・タイトル・メタ情報の再順位付けは既存ロジックを維持。
- sqlite-vecの読込・同期・KNN検索が失敗した場合だけ、既存JavaScript cosine総当たりへ自動フォールバック。

## 運用
- Smart Assist管理画面で、直近検索経路・検索時間・sqlite-vec利用回数・JSフォールバック回数を確認できる。
- テレメトリはアプリ実行中のメモリ内だけに保持し、検索ごとのSQLite書込みは行わない。
- 既存Embeddingの保存形式・共有データ・正本ページは変更しない。

## 期待する状態
- `sqlite-vec利用回数` が増え、`JSフォールバック` が0件に近ければ、通常検索はsqlite-vec経路で動作している。
- フォールバックが増える場合は、sqlite-vecのロード・ベクトル表同期・モデル/次元不一致を確認する。
