# V166 Private Database Restart Fix

Private DBがアプリ再起動後に一覧から消えるように見える問題を修正しました。

## 修正内容

- DB一覧読み込み時に Shared DB と Private DB の両方を必ず読むように強化。
- Private DB保存先を変更した場合でも、従来の自動保存先・旧保存先候補を探索。
- 同じDB IDが複数場所に存在する場合は、更新日時が新しいものを採用。
- Private/Shared切替時は、新しい保存先へ書き込んでから旧保存先を削除する安全順序に変更。
- 削除時は主保存先だけでなく、旧保存先候補に残った同名DBもバックアップへ退避。

## 重要

この修正は、データを消すものではなく「見えなくなっていたPrivate DBを再発見する」ための修正です。
