# V468 — 文書作成モード（最大 2048 トークン）

## 目的
通常の質問回答を軽量に保ったまま、通知文・案内文・FAQ原案・会議録・報告メモなどを、保存前提の完成原稿として作成する。

## UI
AIチャットの「回答」メニューに **文書作成** を追加する。選択時は自動的に以下へ切り替える。

- 長さ: 詳しく
- 読込: 詳細
- 推奨 llama-server ctx: 8192

## 生成制御
- 文書作成モードのみ最大 `2048` トークンを許可する。
- 通常の高速/標準/詳細回答と、手順化の上限は変更しない。
- 出力上限であり、常に2048トークンを生成するわけではない。
- 未確定の日付・金額・連絡先等は推測せず、`【要確認】` として明示する。

## 根拠
参照元がない一般知識の断定は従来どおり抑止する。文書作成でも、ワークスペース内の参照候補または現在ページを根拠に生成する。

## 性能上の注意
2048トークンは通常回答には重すぎるため、常用しない。llama-server常駐モードとctx 8192を推奨する。会社Windows端末では、通常の質問回答より生成時間とメモリ使用量が増える。
