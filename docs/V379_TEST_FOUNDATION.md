# v379 テスト基盤・保存ルール共通化

## 目的
保存・ロック・共有フォルダ整合性の修正を将来の機能追加で壊さないため、Electron/Reactに依存しない保存ルールを共有モジュールへ切り出し、Node標準テストランナーで検証できる状態にします。

## 実行

```bash
npm test
```

## 初期テスト対象
- 保存失敗時の 2秒 / 5秒 / 10秒の再試行計画
- 競合(409)・ロック(423)を自動再試行しない判定
- `commit.json` の writing 状態を同期対象から除外する判定
- 一時ファイル名の一意性

## 今後の追加対象
- ページ/DBロックのTTLと排他取得
- Privateページのコメント・添付保存先
- DB保存の `baseUpdatedAt` 判定
- JST日付条件とDB条件抽出

## Private境界テスト
`scopeBoundary.test.ts` は、Privateページに紐づくデータがShared保存先を選ばないための基本ルールを検証します。添付・コメントのパス決定はこのルールを使います。
